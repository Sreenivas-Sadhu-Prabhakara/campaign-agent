01

The optimal path

No hedging. One stack, one webhook for all of Meta, one drip engine that routes by channel. It's chosen for the shortest distance from zero to a running, deployable app — and it's the code already sitting in this repo's /starter folder.

LayerThe pickWhy this, decisively
AppNext.js 15 + TypeScript + TailwindOne framework for UI, API, and cron. Every AI knows it cold.
DataPrisma + SQLite → PostgresZero-config locally; one line to switch to hosted Postgres in prod.
EmailResendSimplest API, free tier, easy domain auth.
MetaGraph API + WhatsApp Cloud APIFB/IG Lead Ads, WhatsApp, Messenger/IG DMs — all through one app + one webhook.
HostVercel (+ Neon Postgres)Git-push deploy; built-in Cron runs the daily drip for free.

How it flows

Instagram Lead Ad ──┼──▶ Lead ──▶ Day 2  Value      ──▶ opens · clicks

🧭

The one rule that shapes everything: Lead Ads collect a name/email/phone — you nurture those by email or WhatsApp. WhatsApp & DMs can only be started with an approved template (WhatsApp) or inside the 24-hour reply window (DMs). The starter handles both correctly.

04

Run it in 5 commands

These are the exact commands — verified to install, build, and run on a clean machine.

terminal
git clone https://github.com/Sreenivas-Sadhu-Prabhakara/campaign-agent.git
cd campaign-agent/starter

npm install                 # install dependencies
cp .env.example .env        # then add RESEND_API_KEY + EMAIL_FROM
npx prisma db push          # create the local SQLite database
npm run db:seed             # optional: a few demo leads
npm run dev                 # ▶ http://localhost:3000

Then open these three URLs

URLWhat it is
localhost:3000Landing page — submit the form to create a lead
localhost:3000/dashboard?key=YOUR_ADMIN_KEYLeads, conversion, channel breakdown
localhost:3000/api/test-send?key=YOUR_ADMIN_KEY&to=you@email.comSends all 3 email templates to you
✅

YOUR_ADMIN_KEY is the ADMIN_KEY value from your .env. Leads still save even if email isn't configured yet — the welcome send is skipped gracefully.

05

Write the campaign

All wording lives in one file: starter/lib/content.ts. It holds the welcome / value / offer copy for every channel — email subject + body, and the short WhatsApp/DM text. Edit it by hand, or have an AI write it for you with this prompt:

prompt → paste output into lib/content.ts
Rewrite the COPY object in this file for my business.
Business: [WHAT YOU SELL]. Audience: [WHO + their main pain point].
Goal of the sequence: get them to [book a demo / start a trial / reply].
Tone: helpful, concise, no hype.

- welcome (Day 0): thank them, deliver the lead magnet, one soft CTA.
- value (Day 2): one genuinely useful tip that builds trust, light CTA.
- offer (Day 5): a clear, time-bound offer, single CTA.

Keep the exact TypeScript shape (subject, paragraphs[], cta{label,url}, text).
Keep each email body under ~120 words and each WhatsApp text under ~3 lines.
Use the {firstName(l)} helper. Return only the updated COPY object.
🟢

WhatsApp note: the text field is used for in-window replies. To start a WhatsApp conversation (the drip), you also need pre-approved templates named lead_welcome / lead_value / lead_offer in Meta — see section 06.

06

Connect Meta — FB · IG · WhatsApp

All four Meta channels run through one app and one webhook: https://YOUR_APP/api/webhooks/meta. Do this after deploying (section 07) so you have a public HTTPS URL — or use a tunnel (e.g. ngrok) for local testing.

  1. Create a Meta app at developers.facebook.com → Business type. Copy the App Secret into META_APP_SECRET.
  2. Set the webhook. App → Webhooks → callback URL = https://YOUR_APP/api/webhooks/meta, verify token = your META_VERIFY_TOKEN. Subscribe to leadgen, messages, messaging_postbacks.
  3. Facebook & Instagram Lead Ads. Connect your Page, generate a Page access token → META_PAGE_ACCESS_TOKEN. New lead-form submissions now hit your webhook; the app fetches the name/email/phone and starts the drip.
  4. WhatsApp. Add the phone number; copy WHATSAPP_TOKEN + WHATSAPP_PHONE_NUMBER_ID. In WhatsApp → Message Templates, create & submit three templates, each with one body variable {{1}} (the name):
    templates to create
    lead_welcome   "Hi {{1}} 👋 Thanks for signing up! Here's your guide…"
lead_value     "Quick tip {{1}}: following up 3x lifts replies 2–3x…"
lead_offer     "{{1}}, ready to start? Here's your next step…"
  5. Messenger & Instagram DMs. Already covered by the same Page token + webhook. Inbound DMs create a lead and auto-reply within the 24h window.
⚠️

Policy reality: WhatsApp template messages must be approved before they send. Messenger/IG outside the 24-hour window need an approved message tag. The code is built around these rules — don't try to cold-DM users.

07

Go live (Vercel)

From inside starter/:

terminal
npm i -g vercel
vercel            # link/create the project
vercel --prod     # deploy → you get a public https URL
  1. Switch to Postgres. In prisma/schema.prisma change provider = "postgresql", then set DATABASE_URL to a free Neon or Supabase connection string. Run npx prisma db push against it once.
  2. Add env vars in Vercel → Settings → Environment Variables: everything from .env.example. Set APP_URL to your real domain (no trailing slash).
  3. Cron is automatic. vercel.json already schedules the daily drip at 14:00 UTC and Vercel sends your CRON_SECRET for you.
  4. Email deliverability. In Resend, add the SPF / DKIM / DMARC DNS records for your domain. Then score a test at mail-tester.com (aim 9+/10).
  5. Point Meta at production. Use the live URL for the webhook in section 06.

09

Compliance & deliverability

Email — CAN-SPAM / GDPR / CASL

  • ✅ Working unsubscribe in every email (the starter adds it automatically)
  • ✅ Real physical address in the footer (COMPANY_ADDRESS)
  • ✅ SPF / DKIM / DMARC set up; warm a new domain gradually
  • ✅ Consent (opt-in) for EU/Canada contacts — and record it

WhatsApp & Meta messaging

  • ✅ Opt-in is required before WhatsApp outreach; only approved templates start conversations
  • ✅ Free-form WhatsApp/DM replies only within the 24-hour window
  • ✅ Honor opt-outs across every channel (status → unsubscribed)
🧪

Pre-launch: /api/test-send for emails, mail-tester for spam score, and send a real WhatsApp template to your own number.
