The whole app lives in this repo's starter/ folder. Clone it.

cd campaign-agent/starter
✓

Step 2⏱ 4 min💲 free

Install & run

Install dependencies, create a local database (SQLite — no setup, no cost), and start it.

terminal
npm install           # install dependencies
npx prisma db push    # create the local database
npm run dev           # start → http://localhost:3000

Open http://localhost:3000 and submit the form. Then view it in the dashboard (the key is ADMIN_KEY in your .env — defaults work for now):

browser
http://localhost:3000/dashboard?key=change-me-to-something-secret
✓
Checkpoint: the lead you just submitted appears in the dashboard table. (Email won't send yet — that's step 3.)
Next: turn on email ↓
Step 3⏱ 5 min💲 free · 3,000/mo

Turn on email

Resend's free tier sends 3,000 emails/month (100/day) — plenty to start. No credit card.

  1. Sign up at resend.com and create an API key.
  2. Copy the example env file and paste your key in:
terminal
cp .env.example .env
.env (edit these two)
RESEND_API_KEY="re_your_key_here"
EMAIL_FROM="Your Name <onboarding@resend.dev>"   # resend.dev works free for testing

Restart npm run dev, then send all three templates to yourself:

browser
http://localhost:3000/api/test-send?key=change-me-to-something-secret&to=you@email.com
✓
Checkpoint: three emails (welcome / value / offer) land in your inbox, each with an unsubscribe link in the footer.
Next: write your campaign ↓
Step 4⏱ 5 min💲 free

Write your campaign

All wording for every channel lives in one file: lib/content.ts. Edit it by hand, or let a free AI write it. Paste this prompt into Gemini or any assistant:

prompt → paste result into lib/content.ts
Rewrite the COPY object in this file for my business.
Business: [WHAT YOU SELL]. Audience: [WHO + their main pain point].
Goal: get them to [book a demo / start a trial / reply]. Tone: helpful, no hype.

- welcome (Day 0): thank them, deliver the lead magnet, one soft CTA.
- value   (Day 2): one genuinely useful tip, light CTA.
- offer   (Day 5): a clear, time-bound offer, single CTA.

Keep the exact TypeScript shape (subject, paragraphs[], cta{label,url}, text).
Emails under ~120 words; WhatsApp text under 3 lines. Return only the COPY object.
✓
Checkpoint: re-run /api/test-send — the emails now reflect your offer and name.

Step 5⏱ 5 min💲 free

Deploy for $0

Vercel's Hobby plan hosts it free, with a free your-app.vercel.app URL and a built-in cron for the daily drip. Run this in starter/:

terminal
npm i -g vercel
vercel          # link/create the project
vercel --prod   # deploy → returns your public URL

Two quick settings in the Vercel dashboard:

  1. Free database: in prisma/schema.prisma set provider = "postgresql", then create a free Neon database and paste its URL into DATABASE_URL. Run npx prisma db push once against it.
  2. Env vars: add everything from .env.example under Settings → Environment Variables. Set APP_URL to your .vercel.app URL.
✓
Checkpoint: your public URL loads, the form creates a lead, and the welcome email sends. You're live for $0.
Optional: custom domain & better deliverability ›

The free .vercel.app URL works fine to start. When you're ready, add a domain (~$10/yr) and verify it in Resend with SPF / DKIM / DMARC DNS records — then test at mail-tester.com (aim 9+/10). Better inbox placement = more leads.

Step 6⏱ 15 min💲 organic free💲 ads paid

Add Meta channels — optional

Email already works end-to-end. Add WhatsApp, Facebook & Instagram when you want them — all through one webhook: https://YOUR_APP/api/webhooks/meta. Expand only what you need.

🟢 WhatsApp & Messenger / Instagram DM — free ›
  1. Create a Meta app at developers.facebook.com; copy the App Secret → META_APP_SECRET.
  2. Webhooks → callback = your /api/webhooks/meta URL, verify token = your META_VERIFY_TOKEN. Subscribe to leadgen, messages.
  3. WhatsApp → add the number; copy WHATSAPP_TOKEN + WHATSAPP_PHONE_NUMBER_ID. Create & submit 3 templates named lead_welcome / lead_value / lead_offer, each with one {{1}} body variable (the name).

WhatsApp has a free tier of service conversations; you only pay beyond it. Inbound DMs & replies within 24h are free.

🔵🟣 Facebook & Instagram Lead Ads — needs ad budget ›

Connect your Page and add META_PAGE_ACCESS_TOKEN. Lead-form submissions hit your webhook; the app fetches the name/email/phone and starts the drip automatically. Running the ads costs whatever budget you set — the app and webhook are free.

✓
Checkpoint: Meta shows a green tick on the webhook, and a test Lead Ad / WhatsApp message appears as a lead in your dashboard.

No-database variant

Want zero infrastructure — nothing to deploy or provision? Use the Google Sheets + Apps Script version. The Sheet is your storage, Apps Script runs the form & drip free, Gmail sends the emails, and the landing page hosts free on GitHub Pages.

flow
index.html (form) ──POST──▶ Apps Script ──▶ Google Sheet (storage)
                               │
                               ├─▶ welcome email (instant, via Gmail)
                               └─▶ daily trigger → Day 2 / Day 5 follow-ups
Set it up in ~10 minutes (free) ›
  1. New Google Sheet → Extensions → Apps Script.
  2. Paste no-db/Code.gs; edit the CONFIG + COPY blocks for your business.
  3. Run installDailyTrigger once (approve permissions) — this creates the sheet and schedules the daily drip.
  4. Deploy → New deployment → Web app (Execute as: Me · Access: Anyone). Copy the /exec URL.
  5. Paste that URL into no-db/index.html (WEBAPP_URL) and host the page free on GitHub Pages.

Trade-offs: no open/click tracking and no WhatsApp/Meta channels; Gmail caps ~100 sends/day. Perfect for validating an offer at $0 — graduate to the full app when you outgrow it.

🪶
Pick this when: you want email-only, the smallest footprint, and $0 forever. Full code + steps live in no-db/ — view on GitHub.
