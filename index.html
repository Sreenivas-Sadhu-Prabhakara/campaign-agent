Build a custom lead-gen email app with AI

A practical, copy-paste playbook for turning Claude, OpenAI Codex / ChatGPT, Google Gemini, and Microsoft Copilot into a team that builds your email campaign app — and writes the emails that convert.

⚡ Grab the master prompt Pick a tool →

01

What you're building

A lightweight web app that runs lead-generation email campaigns — it captures visitors as leads, nurtures them with an automated email sequence, and shows you what's converting. Here's the engine, part by part:

🎯

Landing page + capture form

A focused page with one job: turn a visitor into an email address.

🗂️

Lead database

Every contact with source, tags, and status — new, engaged, converted, unsubscribed.

✉️

Email templates

Reusable welcome / value / offer emails with merge tags like {{name}}.

⏱️

Drip automation

Day 0 → 2 → 5 follow-ups that warm a cold signup without you lifting a finger.

📊

Tracking

Opens, clicks, and conversions so you know exactly what's working.

📈

Dashboard

Lead count, conversion rate, and your best-performing email at a glance.

Prerequisites

Install these once. If you get stuck, paste the error into your AI — that's exactly what it's for.

  1. Node.js (LTS) — nodejs.org
  2. Git — git-scm.com
  3. VS Code — the most widely supported editor for these tools
  4. An email-sending API key — sign up at Resend / SendGrid / SES
  5. A domain — for production sending + deliverability (you'll add DNS records later)

Then create your project folder:

bash

git init
04

The master prompt

This one prompt works in all four tools. Paste it as your first message. Save it somewhere — you'll reuse it constantly.

master prompt
You are my senior full-stack engineer. Build a custom email-campaign web app
whose goal is to capture and convert more sales leads.

TECH STACK
- Next.js (App Router) + TypeScript + Tailwind CSS
- SQLite via Prisma for local dev (make it easy to switch to Postgres later)
- Resend for sending email (read RESEND_API_KEY from .env)

FEATURES (build incrementally, smallest working version first)
1. Public landing page with a value proposition headline and an email
   capture form (name + email). On submit, save the lead and send a
   welcome email.
2. A "leads" table: id, name, email, source, tags, status
   (new | engaged | converted | unsubscribed), createdAt.
3. Admin dashboard (simple password-protected) showing total leads,
   new this week, conversion rate, and a list of leads.
4. Email templates stored as React/HTML with merge tags ({{name}}).
5. A drip sequence: Day 0 welcome, Day 2 value email, Day 5 offer email.
   Implement as a scheduled job / cron-friendly endpoint.
6. Open & click tracking (tracking pixel + redirect links) and an
   unsubscribe link in every email.

CONSTRAINTS
- Mobile-responsive, accessible, clean modern UI.
- Include input validation and basic spam protection (honeypot field).
- Add a .env.example, a README with run instructions, and seed data.
- Follow email compliance: physical address + unsubscribe in footer.

PROCESS
- Propose the file/folder structure first and wait for my "go".
- Then implement step 1 end-to-end and tell me how to run it before
  moving on. Explain key decisions briefly.
⚙️

Tweak per tool: for an in-editor agent (Copilot/Gemini) the prompt is identical — just paste it into the chat panel. For the no-code route, swap the stack line for "a single self-contained HTML file that posts to Airtable."

05

Build it, tool by tool

Pick a tab. Each shows the fastest path from zero to a running app with that assistant.

C

Claude (Claude Code)

Strongest at building and running the whole project for you.

Option A — Claude Code (recommended, terminal)

  1. Install
    bash
    npm install -g @anthropic-ai/claude-code
  2. Start it in your project folder
    bash
    cd lead-campaign-app
claude
  3. Paste the master prompt. Claude proposes a file structure — type go.
  4. Let it build. It writes files, installs deps, and runs the dev server. Approve commands as prompted.
  5. Iterate by talking to it:
    prompt
    Add A/B testing for subject lines and show open-rate per variant.
The welcome email looks plain — match brand color #1a73e8 and add a CTA button.
Write tests for the lead-capture API and run them.
  6. Make it verify itself:
    prompt
    Run the app, open the landing page, submit a test lead, and confirm
the welcome email is queued.
  7. Deploy: Set this up to deploy to Vercel and walk me through connecting Postgres.

Option B — Claude.ai (browser, no install)

Paste the master prompt at claude.ai; copy files into your editor, or use Artifacts for an instant live preview of the landing page.

Option C — Claude API (make the app itself smart)

Use the Claude API inside your app to auto-write personalized follow-ups per lead. Ask: "Add an endpoint that calls the Claude API to draft a personalized follow-up from the lead's name and source, with prompt caching enabled."

Two products — use both for different jobs.

  • Copilot Studio: build a no-code site chatbot that captures leads conversationally.

The app blueprint

Use this to sanity-check whatever the AI generates.

File structure

tree
lead-campaign-app/
├─ app/
│  ├─ page.tsx                 # Landing page + capture form
│  ├─ api/
│  │  ├─ leads/route.ts        # POST: create lead, send welcome
│  │  ├─ track/open/route.ts   # GET: tracking pixel
│  │  ├─ track/click/route.ts  # GET: click redirect + log
│  │  ├─ unsubscribe/route.ts  # GET: one-click unsubscribe
│  │  └─ cron/drip/route.ts    # Scheduled: send next drip email
│  └─ dashboard/page.tsx       # Admin metrics + lead list
├─ emails/
│  ├─ welcome.tsx              # Day 0
│  ├─ value.tsx                # Day 2
│  └─ offer.tsx                # Day 5
├─ lib/
│  ├─ db.ts                    # Prisma client
│  ├─ mailer.ts               # Resend wrapper + merge tags
│  └─ tracking.ts             # pixel + link helpers
├─ prisma/schema.prisma        # Lead, EmailEvent models
├─ .env.example
└─ README.md

Data model (minimum)

prisma
model Lead {
  id        String   @id @default(cuid())
  name      String?
  email     String   @unique
  source    String?  // e.g. "facebook-ad", "homepage"
  tags      String?  // comma-separated
  status    String   @default("new") // new|engaged|converted|unsubscribed
  createdAt DateTime @default(now())
  events    EmailEvent[]
}

model EmailEvent {
  id        String   @id @default(cuid())
  leadId    String
  lead      Lead     @relation(fields: [leadId], references: [id])
  type      String   // sent|opened|clicked|converted
  emailKey  String   // welcome|value|offer
  createdAt DateTime @default(now())
}

The lead-conversion loop

flow
Visitor → Landing page → Submit form → Lead saved (status: new)
   → Welcome email (Day 0) → opens/clicks tracked (status: engaged)
   → Value email (Day 2) → Offer email (Day 5)
   → Clicks offer / books call → status: converted → notify you
07

Writing the emails

A great app sends mediocre results without great copy. Claude, ChatGPT, Gemini, and M365 Copilot are all strong writers — feed any of them this prompt:

email sequence prompt
Write a 3-email lead-nurture sequence for [YOUR PRODUCT/SERVICE].
Audience: [who they are and their main pain point].
Goal: get them to [book a demo / start a free trial / reply].
Tone: helpful, concise, no hype.

Email 1 (Day 0, Welcome): thank them, set expectations, deliver the
  lead magnet, one soft CTA.
Email 2 (Day 2, Value): teach one useful tip that builds trust, light CTA.
Email 3 (Day 5, Offer): make a clear, time-bound offer with a single CTA.

For each: give 3 subject-line options, preview text, and body with a
{{name}} merge tag. Keep each under 150 words. Plain, scannable formatting.

High-converting email tips

🎯

One CTA per email

Every email asks for exactly one action. Choices kill conversions.

🧪

A/B the subject line

Short, specific, benefit- or curiosity-driven. Test two, keep the winner.

👤

Personalize beyond the name

Reference their source — "Saw you came from our pricing guide…"

📱

Mobile-first

Short paragraphs, one big tappable button. Most opens are on phones.

08

Compliance & deliverability

Ask your AI to bake these in. They protect your sender reputation and keep you legal.

Legal — CAN-SPAM / GDPR / CASL

  • ✅ A working unsubscribe link in every email (honor within 10 days)
  • ✅ A real physical mailing address in the footer
  • ✅ No deceptive subject lines or "from" names
  • ✅ For EU/Canada contacts, get opt-in consent before emailing — and record it

Deliverability — reach the inbox, not spam

  • Set up SPF, DKIM, and DMARC DNS records for your sending domain
  • Warm up a new domain gradually — don't blast thousands on day one
  • Keep lists clean: auto-remove hard bounces and unsubscribes
🧭

Ask your AI: "Walk me through adding SPF/DKIM/DMARC for Resend on my domain."

Test before launch

  • Send tests to yourself across Gmail, Outlook, and mobile
  • Score spamminess at mail-tester.com (aim for 9+/10)
  • Verify the tracking pixel and unsubscribe actually fire
prompt
Add a /api/test-send endpoint that sends each template to my address
with sample merge data, and a script to run it.

Host this guide on GitHub Pages

This page is a single self-contained index.html — no build step. Publish it free in three commands.

  1. Create a repo & push (run inside this folder)
    bash
    git init
git add index.html
git commit -m "Add Lead Engine field guide"
git branch -M main
git remote add origin https://github.com/<you>/lead-engine-guide.git
git push -u origin main
  2. Turn on Pages: on GitHub → Settings → Pages → Source: Deploy from a branch → Branch: main / (root) → Save.
  3. Visit your site at https://<you>.github.io/lead-engine-guide/ (live in ~1 minute).
⚡

One-command alternative (with the GitHub CLI installed):

bash
gh repo create lead-engine-guide --public --source=. --push
gh api -X POST repos/:owner/lead-engine-guide/pages -f source[branch]=main -f source[path]=/
📁

If your file isn't named index.html, rename it so GitHub Pages serves it as the homepage automatically.
